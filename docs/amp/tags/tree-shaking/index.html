  • 2.5X Smaller Angular 2 Applications with Google Closure Compiler

    • Jul 21, 2016
    • post
    In the post “Building an Angular 2 Application for Production” we explored how we can decrease the bundle size of a “Hello world!” application to from about 1.6M (non-minified & uncompressed) to 49K! We did this by using the following techniques: Angular offline template compiler for generating tree-shakable code. Tree-shaking of ES2015 modules with rollup. Bundling of the application. Minification with uglifyjs. Compression with gzip. Although we achieved impressive results we can do even better!

    Progressive Web Applications help us build native-like web apps, thanks to amazing tools such as Service Workers, IndexDB, App Shell etc. Once the browser downloads all the static assets required by our app, the active Service Worker can cache them locally. This way the user may experience slowdown during the initial page load, but each next time she opens the application her experience will be instant!
